# Слой моделей (Model Layer)

Управление **LLM и смежными моделями** без привязки к одному вендору на уровне архитектуры.

## Компоненты

| Компонент | Назначение |
|-----------|------------|
| **LLM Provider** | Конкретный API (OpenAI, Anthropic, локальные и т.д.) |
| **Model Router** | Выбор модели по задаче, стоимости, латентности, политикам |
| **Model Adapter** | Единый интерфейс запроса/ответа, streaming, tool schema |
| **Fallback Model** | Резерв при недоступности или отказе по policy |
| **Embedding Model** | Векторизация для RAG и семантической памяти |
| **Reranker** | Переранжирование кандидатов retrieval |
| **Vision / Speech** | Опционально для мультимодальных сценариев |

## Агрегаторы (возможный вариант)

**OpenRouter** — пример **агрегатора/маршрутизатора** к нескольким моделям; использование — **решение проекта**, не зафиксированное требование MARS v0.

## Статус

Политики роутинга, лимиты токенов и список разрешённых моделей — **будущая** конфигурация окружения и security.

## tool-layer.md

# Слой инструментов (Tool Layer)

Инструменты — **контролируемые** действия агентов во внешнем мире (ФС, Git, API, браузер, shell, MCP).

## Примитивы

- **Tools / Function calling** — схема аргументов, валидация (например JSON Schema / Pydantic).
- **Actions** — высокоуровневые операции, собранные из инструментов.
- **Connectors** — интеграции с внешними системами (Sheets, БД, боты).
- **Tool Registry** — каталог, версии, владельцы, уровни риска.
- **Tool Permissions** — кто и когда может вызывать; pre-approval / post-audit.
- **Sandbox Tools** — изолированное исполнение (контейнер, ограниченный FS).
- **Code execution / Shell / Browser / File / API** — классы инструментов с разным risk profile.
- **MCP (Model Context Protocol)** — стандартизованный доступ к ресурсам и тулам внешних серверов.

## Безопасность (обязательное правило)

**MCP и любое исполнение инструментов** требуют **строгих разрешений**, политик, аудита и при необходимости **одобрения человека**. Не включать MCP-серверы «вслепую» в прод без review.

## Рабочий процесс с Web-GPT

Все **массовые операции с ФС**, генерация структуры проекта и страниц — **через Cursor/Codex по промптам**; Web-GPT не правит файловую систему вручную (см. `06-rules/ai-workflow-rule.md`).
