## ai-workflow-rule.md

# Правило ИИ-workflow: кто трогает файловую систему

## Обязательное правило

**Все** операции:

- создание папок и каталогов;
- создание файлов;
- **массовые** правки;
- правки **структуры** проекта;
- **генерация страниц**;
- создание **системного исходного пакета** документации;
- создание **структуры документации**;

должны выполняться **через Cursor/Codex по промптам** (режим **AGENT** там, где нужны записи на диск).

## Запрещено для Web-GPT вручную

- ручное редактирование ФС «от себя» вместо исполнителя;
- обход Cursor для перечисленных классов операций.

## Разрешено Web-GPT

- планы, спецификации, чеклисты;
- формулировка промптов и критериев приёмки;
- разбор отчётов и сигналов системы.

Исключения (если появятся) должны быть **явно** описаны в project rules и не противоречь уставу безопасности.

# 06-rules/06-rules__chatgpt-working-rules.md

## chatgpt-working-rules.md

# Правила работы Web-GPT в контуре MARS

## Ответы пользователю

- Язык ответов: **русский**.
- Формат: **кратко**, структурировано, **Markdown**.

## Cursor-промпты

- Текст промптов для Cursor: **английский** (если команда не решит иначе в project rules).
- Всегда указывать режим: **AGENT** или **ASK**.
- Всегда указывать **окно/папку** Cursor (workspace path).
- Если открыто **несколько** окон Cursor — явно сказать, **в каком** выполнять шаг.

## Визуальные акценты

Для важных моментов — **крупные заметные маркеры** и **эмодзи** (по договорённости команды), чтобы снизить риск пропуска шага.

## Prompt → execute → report

Если всё прошло штатно — **не** добавлять длинные пояснения в отчётную фазу.

## Файловая система

**Все операции создания папок, создания файлов, массовых правок, правок структуры проекта и генерации страниц выполняются через Cursor/Codex по промптам.**

Web-GPT **не** правит файловую систему вручную. Web-GPT готовит **инструкции, промпты, планы** и **проверяет отчёты**.

## Git на вехах

На **крупных чекпоинтах** Web-GPT добавляет в **начало** следующего Cursor-промпта шаги: **Git backup / commit / push** (в рамках политики репозитория).

# 06-rules/06-rules__cursor-operation-rule.md

## cursor-operation-rule.md

# Правила операций в Cursor

| Режим | Когда использовать |
|-------|-------------------|
| **AGENT** | Любые **записи** в файлы, создание структуры, массовые правки, генерация страниц |
| **ASK** | **Анализ**, чтение, объяснение без записи |

## Язык промптов

Промпты для агента в Cursor — **английский** (если не переопределено локальными правилами репозитория).

## Запреты исполнителя

- **Не** редактировать **`dist`** и прочие **build-артефакты** — только исходники (`src` и т.п.).
- **Не** удалять файлы без **явной** инструкции в задаче/промпте.

## Отчёт

После выполнения — вернуть **список изменённых файлов** и краткий итог.

# 06-rules/06-rules__rule-precedence.md

## rule-precedence.md

# Приоритет правил MARS

При конфликте инструкций порядок старшинства (сверху вниз):

1. **Безопасность / human approval** — ничто не имеет права обходить явный стоп или запрет.
2. **Устав системы** (`system-charter.md`) — цели, ограничения, HITL.
3. **Правила проекта** (репозиторий, регион, стек).
4. **Правила агента** (карточка агента, permissions).
5. **Промпт текущей задачи** — в рамках вышестоящих уровней.
6. **Стилевые предпочтения** — форматирование, краткость (не в ущерб безопасности).

Если неясно, какой уровень применим — сигнал **UNKNOWN** / эскалация человеку.

## Запрет на псевдо-реализацию

Система НЕ имеет права:
- утверждать, что слой реализован, если он только описан
- ссылаться на несуществующий runtime
- имитировать работу агентов, если это не явно указано

Всегда разделять:
- архитектура (план)
- реализация (факт)
