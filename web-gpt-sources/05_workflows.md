# 04-workflows/04-workflows__execution-rules.md

## execution-rules.md

# Правила исполнения: prompt → execute → report

## Стиль

1. **Prompt** — чёткий, с контекстом окна Cursor, режимом **AGENT** или **ASK**, языком промпта **английский** (если так зафиксировано проектными правилами для Cursor).
2. **Execute** — агент или человек в Cursor выполняет действия; для ФС — **AGENT**.
3. **Report** — кратко: что сделано, список изменённых файлов, сигналы (**UNKNOWN**, **GIT CHECKPOINT NEEDED**, …).

## Запреты

- Не смешивать **анализ** и **массовые правки** в одном неявном шаге без режима.
- Не скрывать **ошибки** за длинными объяснениями.

## Связь с Git

На **major checkpoint** Web-GPT добавляет в **верх** следующего Cursor-промпта шаги: backup / commit / push (по политике репозитория).

## task-contract.md

# Формальный контракт задачи (Task Contract)

Поля, которые **планировщик** и **оркестратор** используют для недвусмысленного исполнения.

| Поле | Описание |
|------|----------|
| **id** | Уникальный идентификатор задачи |
| **title** | Краткое название |
| **goal** | Измеримая цель |
| **scope_in** | Что явно входит |
| **scope_out** | Что явно не входит |
| **inputs** | Файлы, ссылки, данные |
| **outputs** | Ожидаемые артефакты (пути, форматы) |
| **acceptance_criteria** | Критерии приёмки |
| **constraints** | Сроки, стек, запреты (например не трогать `dist`) |
| **risk_level** | low / medium / high |
| **required_agents** | Предполагаемые роли |
| **hitl_gates** | Где нужен человек |
| **signals** | Уже известные системные сигналы (если есть) |

Без заполненного контракта **критические** ветки не запускаются автоматически.

# 04-workflows/04-workflows__workflow-standard.md

## workflow-standard.md

# Стандартный workflow MARS

Базовый цикл, совместимый с наследием **prompt → execute → report** и расширенный контролем качества и логами.

## Шаги

1. **Task** — формулировка цели и контракта (`task-contract.md`).
2. **Context** — сбор контекста (память, реестр проекта, релевантные файлы).
3. **Plan** — планировщик + при необходимости HITL.
4. **Prompt** — финальный промпт для исполнителя (**Cursor: промпты на английском**, режим AGENT/ASK — см. правила).
5. **Execute via Agent** — исполнение через выбранного агента и Cursor/Codex для ФС/кода.
6. **Report** — краткий отчёт; без длинных простыней, если всё ок.
7. **Validate** — Validator Agent или чеклист.
8. **Log** — запись в lifecycle / audit (когда инфраструктура будет).
9. **Git checkpoint** — при сигналах **GIT CHECKPOINT NEEDED** или крупных изменениях: шаги commit/push **в начале** следующего Cursor-промпта (по указанию Web-GPT).

## Явное правило по ФС

Создание папок, массовое создание файлов, генерация страниц, правка структуры проекта — **только через Cursor/Codex по промптам**; Web-GPT не выполняет ручное редактирование файловой системы.
