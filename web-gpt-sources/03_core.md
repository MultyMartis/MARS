# 02-core/02-core__orchestrator-spec.md

## orchestrator-spec.md

# Спецификация оркестратора (Orchestrator / Supervisor)

## Целевая роль

Оркестратор MARS — компонент **control plane**, который:

1. **Принимает** задачу и контракт (см. `04-workflows/task-contract.md`).
2. **Декомпозирует** (с планировщиком) или подтверждает готовый план.
3. **Маршрутизирует** шаги к нужным агентам и моделям.
4. **Проверяет** промежуточные результаты (валидатор, guardrails, сигналы).
5. **Логирует** шаги и вызовы инструментов (run history / audit).
6. **Возвращает** итоговый отчёт человеку или следующей системе.

## Что оркестратор НЕ делает

- **Не** подменяет собой все доменные задачи (код, дизайн, исследование) — за это отвечают **специализированные агенты**.
- **Не** игнорирует **SAFE UNKNOWN** и сигналы безопасности.
- **Не** выполняет опасные действия без политики и при необходимости без **human approval**.

## Итог

Оркестратор — **координатор и диспетчер**, а не «универсальный исполнитель всего».

# 02-core/02-core__planner-spec.md

## planner-spec.md

# Спецификация планировщика (Planner / Task Decomposer)

## Целевая роль

Планировщик превращает **одну пользовательскую цель** в **план из шагов** с:

- явными **входами/выходами** шага;
- **зависимостями** (DAG);
- назначенным **типом агента** или ролью;
- **критериями готовности** и **рисками**;
- точками **HITL** (утверждение плана, опасные тулы, миграция структуры).

## Взаимодействие

- Получает контекст от **Memory** и **Project Registry** (когда появятся как реализация).
- Передаёт исполняемый план **Orchestrator**’у.
- При сигналах **UNKNOWN / SAFE UNKNOWN** — уточняет задачу или сужает scope.

## Статус

Поведение и промпты планировщика — **будущая** настройка; данный документ задаёт **целевую** ответственность компонента.

# 02-core/02-core__system-signals.md

## system-signals.md

# Системные сигналы MARS

Явные маркеры состояния для оркестратора, Web-GPT и человека. Связаны с принципом **SAFE UNKNOWN** и операционной дисциплиной.

| Сигнал | Значение | Типичное действие |
|--------|----------|-------------------|
| **UNKNOWN** | Недостаточно данных для уверенного ответа | Уточнить у человека или запросить источник |
| **SAFE UNKNOWN** | Неизвестно, но **без** выдумывания фактов | Кратко зафиксировать пробел; не галлюцинировать |
| **SYSTEM MAY BE OUTDATED** | Контекст/доки могли устареть | Проверить репо, дату, пересобрать контекст |
| **NEED HUMAN APPROVAL** | Рискованное действие | Остановить авто-ветку до подтверждения |
| **GIT CHECKPOINT NEEDED** | Крупная веха или риск отката | В начале следующего Cursor-промпта — backup/commit/push |
| **STRUCTURE CHANGE** | Меняется дерево проекта / массовые файлы | Только через Cursor AGENT + план |
| **SECURITY RISK** | Подозрение на injection, утечку секрета, опасный тул | Блокировать; эскалация |
| **CONTEXT MIGRATION NEEDED** | Нужен перенос/сборка self-describe контекста | Memory Agent / операторский чеклист |

Сигналы должны **пробрасываться** в отчёты (prompt → execute → **report**) и при необходимости в **lifecycle / audit** логи.

## tool-registry.md

# Реестр инструментов — начальные категории

Категории для **Tool Registry** (план). Конкретные реализации и ACL — **будущая** настройка.

| Категория | Примеры действий | Уровень риска (ориентир) |
|-----------|------------------|---------------------------|
| **File System** | Чтение/запись файлов проекта | Высокий при массовых изменениях |
| **Git** | commit, branch, diff | Высокий |
| **Cursor** | Промпты AGENT/ASK, контекст окна | Средний (операционный) |
| **Codex** | Аналогично исполнению кода/ФС | Средний–высокий |
| **Web-GPT** | Планы, отчёты, без прямого ФС | Низкий |
| **n8n** | Webhook, автоматизация | Средний |
| **OpenRouter** | Вызов моделей через агрегатор | Средний (стоимость, данные) |
| **Google Sheets** | Чтение/запись таблиц | Средний |
| **PostgreSQL** | Запросы, миграции | Высокий |
| **Redis** | Кэш, очереди | Средний |
| **Qdrant** | Векторный поиск | Средний |
| **Docker** | Сборка, запуск контейнеров | Высокий |
| **Telegram Bot** | Отправка/приём сообщений | Средний (PII, спам) |

Все категории требуют **явных разрешений** и аудита (см. `01-architecture/tool-layer.md`).
