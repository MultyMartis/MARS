# Agent Builder (конструктор агентов)

## Роль

**Agent Builder** — агент-архитектор, который по постановке задачи:

- предлагает **нового агента** или обновление существующего;
- заполняет **стандартную карточку** (`agent-standard.md`);
- согласовывает **инструменты и permissions** с политикой безопасности;
- вносит запись в **Agent Registry** (через процесс: промпт Cursor → изменение файлов реестра, не ручное редактирование Web-GPT).

## Наследие

Соответствует идее **Factory Engineer** из «Мой первый ИИ system», переименованной в стандартные термины MAS.

## Ограничения

Не раздаёт «суперправа» без обоснования; при **SECURITY RISK** или **NEED HUMAN APPROVAL** — останавливается.

# Стандарт карточки агента MARS

Каждый агент в **Agent Registry** описывается единым шаблоном (на русском в документации проекта).

## Обязательные поля

| Поле | Описание |
|------|----------|
| **name** | Уникальное имя агента |
| **type** | Категория (Coding, Validator, Research, …) |
| **purpose** | Зачем существует агент (1–3 предложения) |
| **responsibilities** | Что делает и чего не делает |
| **inputs** | Какие артефакты и контракты принимает |
| **outputs** | Что возвращает (формат, ссылки на файлы) |
| **tools** | Разрешённые инструменты / MCP |
| **permissions** | Уровни доступа, pre-approval |
| **limitations** | Границы компетенции, типичные failure mode |
| **workflow** | В каких workflow участвует по умолчанию |
| **validation** | Как проверяется результат (свой + внешний Validator) |
| **changelog** | История изменений карточки и поведения |

## Примечание

Карточка — **живой документ**; изменения проходят через review (человек + Validator при необходимости).

# Gulp Frontend Agent (наследие → специалист MARS)

## Ценность для MARS

Агент **статического фронтенда** на проверенном legacy-стеке: быстрая сборка страниц, переиспользуемые секции, предсказуемый вывод без тяжёлого SPA-фреймворка.

## Стек (наследие, целевой профиль)

- **HTML**, **SCSS**, **JS** без обязательных фреймворков
- **Gulp**, **gulp-file-include**
- **jQuery** — опционально
- **Swiper**, **Fancybox**, **Inputmask**

## Правило source-first

**Редактировать только `src`**, **никогда** не править вручную артефакты **`dist`** / build-выход: сборка генерирует результат.

## Архитектура страниц

- **Переиспользуемые секции и компоненты** (include/partials), единые стили и токены где возможно.
- Согласование с **workflow**: prompt → execute (Cursor AGENT) → report → validate.

## Миграция в MARS

Статус: **draft / planned migration** — спецификация и карточка агента в реестре; исполняемый агент как код — **будущая** фаза (см. дорожную карту, фаза Gulp).

## Связь с терминологией

В MARS это **Coding Agent / Frontend Specialist Agent** с явным подпрофилем **Gulp**.

# Validator Agent (валидация и guardrails)

## Роль

**Validator Agent** выполняет **независимые проверки** результата работы других агентов:

- соответствие **task contract** и критериям готовности;
- **SAFE UNKNOWN**: нет ли выдуманных фактов при отсутствии источников;
- базовые **security** проверки (секреты в diff, опасные команды);
- согласованность со **структурой проекта** и правилами (например, не трогать `dist`).

## Наследие

Функционально близок к **FlyCheck** в legacy: «проверка перед выдачей».

## Входы / выходы

- **Входы:** отчёт исполнителя, diff, ссылки на тесты, сигналы системы.
- **Выходы:** `PASS` / `FAIL` / `NEED CHANGES`, список замечаний, при необходимости сигналы **UNKNOWN**, **GIT CHECKPOINT NEEDED**.

## Статус

Поведение как **автоматизированного** агента — **планируемая** реализация; до неё валидатор может быть **чеклистом** Web-GPT + человек.
