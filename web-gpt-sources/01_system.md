# Устав (Charter) MARS

## Цели

- Построить **управляемую** мультиагентную систему с ясным control plane и воспроизводимыми workflow.
- Обеспечить **качество** (валидация, guardrails, evals) и **прослеживаемость** (логи, аудит, история запусков).
- Сохранить проверенные идеи legacy («Мой первый ИИ system») в **нормализованном** виде.

## Ограничения

- Не заявлять о **фейковом** статусе внедрения: план vs факт разделять явно.
- **Персональные данные** (для РФ): приоритет хранения на **российской инфраструктуре** или явно согласованных решениях (см. `storage-layer.md`, `storage-adapters.md`).
- **Инструменты и MCP** — только с явными разрешениями и политиками (см. `tool-layer.md`, `security-guardrails-layer.md`).

## Правила безопасности

- Вход/выход: валидация, снижение prompt injection, защита секретов.
- **Критические действия** (структура репозитория, удаления, прод-деплой, доступ к секретам) — через **human-in-the-loop** и явные сигналы системы.
- Принцип **SAFE UNKNOWN**: при неопределённости — не галлюцинировать, а маркировать неизвестность и эскалировать при необходимости.

## Human-in-the-loop

Человек:

- утверждает **планы** и **рискованные** шаги;
- разрешает **инструменты** с побочными эффектами;
- подтверждает **миграции контекста** и крупные **изменения структуры**;
- владеет **Git checkpoint** на важных вехах.

Автоматизация MARS **расширяет** человека, а не заменяет его в точках принятия ответственности.

# 00-system/00-system__system-identity.md

# Идентичность системы MARS

## Официальное название

**MARS** — **Multi-Agent Runtime System** (мультиагентная среда исполнения).

**Русское имя:** **Марс**.

## Что такое MARS

MARS — это **новый чистый проект мультиагентной системы (MAS)**, спроектированный на основе **стандартной мультиагентной архитектуры** (оркестрация, специализированные агенты, инструменты, память, наблюдаемость) и **улучшенный выводами из наследия** условно названного legacy: «Мой первый ИИ system».

MARS **не является** копией legacy-кода «как есть»; это **целевая модель** и **документационно-исходная база** для последующей загрузки в проект Web-GPT и поэтапной реализации.

## Позиционирование

| Аспект | Описание |
|--------|----------|
| Тип | Мультиагентная runtime-система с явным control plane |
| Наследие | Идеи Main Brain, реестра агентов, памяти, логов жизненного цикла — **сохраняются концептуально**, терминология — **нормализуется** |
| Статус реализации | Документ описывает **целевую систему**; фактический статус внедрения см. дорожную карту и реестры без приписывания несуществующих релизов |

## Связь с Web-GPT

Данный пакет — **локальный исходный пакет документации** для Web-GPT. Он задаёт язык, слои, правила и границы ответственности до появления отдельного Cursor runtime starter и кода приложения.

# 00-system/00-system__system-manifest.md

## system-manifest.md

# Манифест MARS

## Назначение

MARS предназначен для **координации нескольких специализированных ИИ-агентов** под единым control plane: постановка задач, планирование, маршрутизация к агентам и инструментам, контроль качества, память, безопасность и наблюдаемость — с **обязательным участием человека** там, где это критично.

## Область (scope)

- **Входит:** архитектура, термины, спецификации оркестратора и планировщика, реестры агентов и инструментов, правила работы Web-GPT ↔ Cursor, память и RAG (как целевые слои), миграция идей из «Мой первый ИИ system», дорожная карта фаз.
- **Не входит в этот пакет:** исполняемый код приложения, инициализация Git, установка зависимостей (явно вне scope данного артефакта).

## Принципы

1. **Стандартная MAS-терминология** — без подмены понятий самодельными именами без маппинга.
2. **SAFE UNKNOWN** — при нехватке данных система не выдумывает; явные сигналы и запрос к человеку.
3. **Разделение ролей:** Web-GPT готовит инструкции и промпты; **изменение ФС и массовые правки — только через Cursor/Codex** по промптам (см. правила в `06-rules/`).
4. **Наследие без хаоса** — переносим практики (реестры, трекеры, Git-чекпоинты), не копируем неструктурированный legacy.

## Ожидаемые результаты (outputs)

- Единый **язык системы** для команды и агентов.
- **Спецификации** для оркестратора, планировщика и карты возможностей.
- **Реестры** агентов и инструментов (черновик/план до появления реальной реализации).
- **Правила** приоритета и работы с Cursor/Web-GPT.
- **Основа** для последующей загрузки в репозиторий Web-GPT и пилотного проекта.

Статус каждого результата по фазам — в `08-roadmap/` и `phases.md`, без заявлений о уже завершённой промышленной реализации, если это не зафиксировано отдельно.

# Карта терминологии: legacy → MARS / стандарт MAS

Сопоставление пользовательских и legacy-терминов условной системы **«Мой первый ИИ system»** со **стандартными** терминами MARS и индустрии.

| Legacy / пользовательский термин | Стандартные термины MARS |
|----------------------------------|---------------------------|
| Main Brain | **Control Plane** / **Orchestrator** / **Supervisor** |
| Factory Engineer | **Agent Builder** / **Architect Agent** |
| FlyCheck | **Validator** / **Guardrail** / **Health Check Agent** |
| Agent Registry | **Agent Registry** / **Agent Catalog** |
| System Signals | **Guardrails** / **Alerts** / **System Signals** |
| Self-Describe | **Introspection** / **Self-Description** / **System Context** |
| Project Registry | **Project Registry** / **System Registry** |
| Lifecycle Log | **Audit Log** / **Lifecycle Log** / **Run History** |
| Workflow Tracker | **Workflow State** / **Workflow Tracker** |
| Gulp Frontend Agent | **Coding Agent** / **Frontend Specialist Agent** (сохраняем специализацию Gulp/статика в спецификации агента) |

## Примечание

В документах MARS предпочтительно использовать **правый столбец**. Левый столбец допустим в скобках при миграции или обучении команды.
