# Правила git-checkpoint’ов (MARS)

**Статус:** зафиксировано в **документации** (как **planned** культура процесса).  
**Связь с архитектурой:** относится к слою **Workflow** и к общей дисциплине **Control Plane / релизов** — не к коду рантайма.

## Цель

Свести **git** к **редким осмысленным точкам** (milestone’ы), а не к фиксации после каждого изменения в файлах. По умолчанию **коммитов и пуша нет** — агент **не** предлагает checkpoint как норму.

## Когда **следует** делать git-checkpoint (commit, при необходимости push)

Только если выполняется смысловой критерий:

1. **Новый системный слой** — появляется существенная новация уровня подсистемы: Control Plane, система агентов, памяти, хранения и т.п. (не пустая папка-заглушка, а осознанно принятая **версия** границы/контракта слоя).
2. **Устойчивый архитектурный milestone** — согласованный итог, который меняет то, *как* устроен MARS (не правка опечатки).
3. **Перед рискованным или крупным изменением** — рефакторинг, перестройка структуры, массовая перенумерация/слияние модулей: checkpoint **до** внесения риска.
4. **Перед внешними интеграциями** — n8n, OpenRouter, адаптеры хранилищ, новые внешние API: зафиксировать известно-хорошее состояние.
5. **Перед deployment или пилотным прогоном** — снимок, с которого можно откатываться.

*Уточнение:* мелкие пакеты doc-only **не** являются сами по себе milestone’ом, если нет отдельного решения “зафиксировать как контрольную точку”.

## Когда **не** поднимается вопрос checkpoint’а (по умолчанию)

- Создание **папок** и **заглушек** (placeholder) без смены смысла релиза.
- Редактирование **документации** (в т.ч. `web-gpt-sources/`, README одного слоя).
- **Небольшие** правки конфигов и **правил** (в т.ч. Cursor, AGENTS) без milestone’а.
- **Мелкие** структурные подправки без риска и без нового слоя.

## Сигнал **GIT CHECKPOINT NEEDED**

- Это **редкий, высокоуровневый** сигнал: *«сейчас уместен осознанный commit (и при договорённости — push)»*.
- **Не** является ответом по умолчанию и **не** выдаётся после каждой файловой операции.
- **Default:** сигнал **не** указывать; `git status` в отчёте **без** призыва к коммиту, пока нет критерия выше.
- Допустимо писать **NO GIT CHECKPOINT** или просто **omit**, если сценарий явно “док/мелочь/черновик”.

## Согласованность

Cursor/агент следует сопоставлять поведение с этим файлом и **не** подменять *«есть несохранённый смысл в git»* на *«нужен commit»*.

**SAFE UNKNOWN:** если граница *major vs minor* не ясна — явно сказать и предложить выбор пользователю, а не тянуть к автоматическому checkpoint’у.
